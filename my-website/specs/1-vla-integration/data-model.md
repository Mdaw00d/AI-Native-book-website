# Data Model: VLA Integration

**Feature:** Vision-Language-Action (VLA) Integration
**Created:** 2025-12-07
**Purpose:** Define data structures for VLA pipeline components

## Overview

This document defines the key data entities flowing through the VLA pipeline:
Voice Command → Text → Task Plan → ROS Actions → Perception → Manipulation

## Core Entities

### 1. VoiceCommand

**Description:** Captured audio input from user microphone

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `audio_data` | `bytes` | Raw audio samples | 16kHz PCM, mono |
| `duration` | `float` | Recording duration in seconds | 0.1 - 30.0 |
| `timestamp` | `datetime` | When recording started | ISO 8601 format |
| `sample_rate` | `int` | Audio sample rate | Default: 16000 Hz |
| `channels` | `int` | Number of audio channels | Default: 1 (mono) |

**State Transitions:**
```
RECORDING → RECORDED → TRANSCRIBED → PROCESSED
```

**Validation Rules:**
- `duration` must be at least 0.1s to avoid empty recordings
- `sample_rate` must be 16kHz or 48kHz (Whisper compatible)
- `audio_data` must not be empty

---

### 2. TranscribedCommand

**Description:** Text output from speech-to-text conversion

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `text` | `str` | Transcribed command text | 1-500 characters |
| `confidence` | `float` | Transcription confidence score | 0.0 - 1.0 |
| `language` | `str` | Detected language code | ISO 639-1 (e.g., "en") |
| `timestamp` | `datetime` | When transcription completed | ISO 8601 format |
| `source_audio_id` | `str` | Reference to original VoiceCommand | UUID |
| `processing_time` | `float` | Transcription latency (seconds) | Measured |

**Validation Rules:**
- `text` must not be empty after whitespace trimming
- `confidence` < 0.5 triggers warning to user
- `language` must be supported by downstream LLM (default: "en")

---

### 3. TaskPlan

**Description:** Structured sequence of robot tasks generated by LLM

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `tasks` | `List[Task]` | Ordered list of tasks | 1-20 tasks |
| `feasibility` | `FeasibilityAssessment` | Plan feasibility evaluation | Required |
| `metadata` | `PlanMetadata` | Generation metadata | Required |

**Nested Types:**

#### Task
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `task_id` | `int` | Unique task identifier | 1-N |
| `task_type` | `TaskType` | Type of task | Enum: NAVIGATE, DETECT, MANIPULATE, INSPECT |
| `parameters` | `Dict[str, Any]` | Task-specific parameters | Type-dependent |
| `preconditions` | `List[int]` | task_ids that must complete first | References valid task_ids |
| `expected_duration` | `float` | Estimated time (seconds) | > 0 |

#### FeasibilityAssessment
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `is_feasible` | `bool` | Overall feasibility | Required |
| `confidence` | `float` | Confidence in assessment | 0.0 - 1.0 |
| `warnings` | `List[str]` | Potential issues | Optional |

#### PlanMetadata
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `llm_model` | `str` | Model used for generation | e.g., "gpt-4" |
| `prompt_tokens` | `int` | Input tokens consumed | Measured |
| `completion_tokens` | `int` | Output tokens generated | Measured |
| `generation_time` | `float` | LLM latency (seconds) | Measured |
| `schema_version` | `str` | Task plan schema version | Semantic versioning |

**State Transitions:**
```
GENERATING → GENERATED → VALIDATED → APPROVED/REJECTED → EXECUTING → COMPLETED/FAILED
```

**Validation Rules:**
- `task_id` must be unique within plan
- `preconditions` must not create circular dependencies
- All `task_type` values must have corresponding executors
- `feasibility.is_feasible == False` → plan rejected before execution

---

### 4. NavigationGoal

**Description:** Target pose for robot navigation

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `target_pose` | `Pose3D` | Goal position and orientation | Required |
| `timeout` | `float` | Maximum execution time (seconds) | 10 - 300 |
| `tolerance` | `Tolerance` | Acceptable position/orientation error | Required |

**Nested Types:**

#### Pose3D
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `position` | `Tuple[float, float, float]` | [x, y, z] in meters | Robot frame |
| `orientation` | `Tuple[float, float, float, float]` | [x, y, z, w] quaternion | Normalized |

#### Tolerance
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `position` | `float` | Position tolerance (meters) | 0.01 - 1.0 |
| `orientation` | `float` | Orientation tolerance (radians) | 0.01 - 0.5 |

**Validation Rules:**
- `orientation` quaternion must be normalized (magnitude = 1.0)
- `position` must be within workspace bounds (defined in environment config)
- `timeout` must be reasonable for distance traveled (heuristic: >10s per meter)

---

### 5. ObjectDetection

**Description:** Detected object with 2D and 3D localization

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `class_name` | `str` | Object class label | From COCO dataset |
| `confidence` | `float` | Detection confidence | 0.0 - 1.0 |
| `bbox_2d` | `BoundingBox2D` | Image-space bounding box | Required |
| `pose_3d` | `Pose3D` | 3D position and orientation | Required if depth available |
| `timestamp` | `datetime` | When detection occurred | ISO 8601 format |

**Nested Types:**

#### BoundingBox2D
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `x_min` | `int` | Left edge (pixels) | 0 <= x_min < image_width |
| `y_min` | `int` | Top edge (pixels) | 0 <= y_min < image_height |
| `x_max` | `int` | Right edge (pixels) | x_min < x_max <= image_width |
| `y_max` | `int` | Bottom edge (pixels) | y_min < y_max <= image_height |

**Validation Rules:**
- `confidence` < 0.5 → detection filtered out (configurable threshold)
- `bbox_2d` must have non-zero area
- `pose_3d.position.z` (depth) must be positive

---

### 6. ManipulationGoal

**Description:** Target object and grasp parameters

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `target_object` | `str` | Object class name to grasp | Must match detected object |
| `grasp_pose` | `Pose3D` | Pre-grasp approach pose | Required |
| `grasp_type` | `GraspType` | Type of grasp to execute | Enum: TOP, SIDE, PINCH |
| `place_pose` | `Optional[Pose3D]` | Where to place object | Optional (for pick-only) |

**Validation Rules:**
- `target_object` must exist in recent detections (within 5s)
- `grasp_pose` must be reachable (IK solvable)
- `place_pose` if provided must be within workspace

---

### 7. ExecutionResult

**Description:** Outcome of a task execution

**Fields:**
| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `task_id` | `int` | Reference to executed task | Must exist in plan |
| `status` | `ExecutionStatus` | Outcome status | Enum: SUCCEEDED, FAILED, CANCELLED, TIMEOUT |
| `execution_time` | `float` | Actual time taken (seconds) | Measured |
| `error_message` | `Optional[str]` | Error details if failed | Required if status=FAILED |
| `timestamp` | `datetime` | When execution completed | ISO 8601 format |

**State Transitions:**
```
PENDING → EXECUTING → SUCCEEDED/FAILED/CANCELLED/TIMEOUT
```

**Validation Rules:**
- `status == FAILED` → `error_message` must be provided
- `execution_time` must be > 0
- `timestamp` must be after task start time

## Enumerations

### TaskType
```python
class TaskType(Enum):
    NAVIGATE = "navigate"       # Move to a location
    DETECT = "detect"           # Perceive objects
    MANIPULATE = "manipulate"   # Pick/place operations
    INSPECT = "inspect"         # Visual inspection
```

### GraspType
```python
class GraspType(Enum):
    TOP = "top"         # Vertical top-down grasp
    SIDE = "side"       # Horizontal side grasp
    PINCH = "pinch"     # Two-finger pinch
```

### ExecutionStatus
```python
class ExecutionStatus(Enum):
    SUCCEEDED = "succeeded"
    FAILED = "failed"
    CANCELLED = "cancelled"
    TIMEOUT = "timeout"
```

## Entity Relationships

```
VoiceCommand 1──→1 TranscribedCommand
TranscribedCommand 1──→1 TaskPlan
TaskPlan 1──→* Task
Task 1──→0..1 NavigationGoal
Task 1──→0..* ObjectDetection
Task 1──→0..1 ManipulationGoal
Task 1──→1 ExecutionResult
```

## Persistence & Serialization

**Storage Format:** JSON (human-readable, debugging-friendly)

**Example TaskPlan Serialization:**
```json
{
  "tasks": [
    {
      "task_id": 1,
      "task_type": "navigate",
      "parameters": {
        "target": "kitchen",
        "location": [2.5, 1.0, 0.0]
      },
      "preconditions": [],
      "expected_duration": 15.0
    },
    {
      "task_id": 2,
      "task_type": "detect",
      "parameters": {
        "object_class": "cup",
        "color": "red"
      },
      "preconditions": [1],
      "expected_duration": 3.0
    }
  ],
  "feasibility": {
    "is_feasible": true,
    "confidence": 0.92,
    "warnings": ["Kitchen location approximate"]
  },
  "metadata": {
    "llm_model": "gpt-4",
    "prompt_tokens": 245,
    "completion_tokens": 89,
    "generation_time": 1.3,
    "schema_version": "1.0.0"
  }
}
```
