# URDF for Humanoids

This chapter introduces URDF (Unified Robot Description Format), the standard format for describing robot kinematics and dynamics in ROS. We'll focus on modeling humanoid robots, understanding joints, links, sensors, and how to visualize and simulate humanoid structures.

## Introduction to URDF

URDF is an XML-based format used to describe robot models in ROS and ROS 2. It defines:

*   **Links**: Rigid body parts of the robot (e.g., torso, upper arm, lower arm, head)
*   **Joints**: Connections between links that define how they move relative to each other
*   **Sensors**: Cameras, lidars, IMUs attached to the robot
*   **Visual and Collision Geometry**: 3D meshes or primitive shapes for rendering and collision detection
*   **Inertial Properties**: Mass, center of mass, and inertia tensors for physics simulation

## Basic URDF Structure

A URDF file is structured hierarchically, with a root link and subsequent links connected via joints. Here's a minimal example:

```xml title="simple_robot.urdf"
<?xml version="1.0"?>
<robot name="simple_robot">
  <!-- Base Link -->
  <link name="base_link">
    <visual>
      <geometry>
        <box size="0.5 0.5 0.1"/>
      </geometry>
      <material name="blue">
        <color rgba="0 0 0.8 1"/>
      </material>
    </visual>
  </link>

  <!-- Upper Link -->
  <link name="upper_link">
    <visual>
      <geometry>
        <cylinder radius="0.05" length="0.3"/>
      </geometry>
      <material name="red">
        <color rgba="0.8 0 0 1"/>
      </material>
    </visual>
  </link>

  <!-- Joint connecting base and upper link -->
  <joint name="base_to_upper" type="revolute">
    <parent link="base_link"/>
    <child link="upper_link"/>
    <origin xyz="0 0 0.15" rpy="0 0 0"/>
    <axis xyz="0 0 1"/>
    <limit lower="-1.57" upper="1.57" effort="10" velocity="1.0"/>
  </joint>
</robot>
```

### Key Elements:

*   **`<robot>`**: Root element with a name attribute
*   **`<link>`**: Defines a rigid body
*   **`<visual>`**: Visual representation (geometry, material)
*   **`<joint>`**: Connects two links with a specific joint type
*   **`<origin>`**: Position (xyz) and orientation (roll, pitch, yaw) of the joint
*   **`<axis>`**: Axis of rotation or translation
*   **`<limit>`**: Joint limits (for revolute and prismatic joints)

## Joint Types in URDF

URDF supports several joint types, essential for humanoid modeling:

| Joint Type | Description | Use Case |
|------------|-------------|----------|
| **fixed** | No movement, rigidly attached | Attaching sensors, connecting torso segments |
| **revolute** | Rotates around an axis with limits | Elbows, knees, shoulders, hips |
| **continuous** | Rotates around an axis without limits | Wheels (though rare in humanoids) |
| **prismatic** | Translates along an axis with limits | Telescoping mechanisms |
| **floating** | 6 DOF (3 translation, 3 rotation) | Root link in simulation |
| **planar** | 2D movement (x, y, rotation around z) | Rarely used in humanoids |

For humanoids, **revolute** joints are most common for articulating limbs and the head.

## Humanoid Robot Structure

A typical humanoid robot has the following kinematic structure:

```
                    [head]
                      |
                  [neck joint]
                      |
      [left_shoulder]---[torso]---[right_shoulder]
            |                           |
      [left_elbow]                [right_elbow]
            |                           |
      [left_wrist]                [right_wrist]
            |                           |
      [left_hand]                 [right_hand]

                  [pelvis]
            /                \
     [left_hip]          [right_hip]
         |                    |
    [left_knee]          [right_knee]
         |                    |
    [left_ankle]         [right_ankle]
         |                    |
    [left_foot]          [right_foot]
```

## Example: Simple Humanoid Torso and Arm

Let's model a humanoid torso with one arm:

```xml title="humanoid_arm.urdf"
<?xml version="1.0"?>
<robot name="humanoid_arm">

  <!-- Torso -->
  <link name="torso">
    <visual>
      <geometry>
        <box size="0.3 0.2 0.5"/>
      </geometry>
      <material name="grey">
        <color rgba="0.5 0.5 0.5 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <box size="0.3 0.2 0.5"/>
      </geometry>
    </collision>
    <inertial>
      <mass value="5.0"/>
      <inertia ixx="0.1" ixy="0.0" ixz="0.0"
               iyy="0.1" iyz="0.0" izz="0.05"/>
    </inertial>
  </link>

  <!-- Upper Arm -->
  <link name="upper_arm">
    <visual>
      <geometry>
        <cylinder radius="0.04" length="0.3"/>
      </geometry>
      <origin xyz="0 0 -0.15" rpy="0 0 0"/>
      <material name="skin">
        <color rgba="0.9 0.8 0.7 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <cylinder radius="0.04" length="0.3"/>
      </geometry>
      <origin xyz="0 0 -0.15" rpy="0 0 0"/>
    </collision>
    <inertial>
      <mass value="1.0"/>
      <origin xyz="0 0 -0.15" rpy="0 0 0"/>
      <inertia ixx="0.01" ixy="0.0" ixz="0.0"
               iyy="0.01" iyz="0.0" izz="0.001"/>
    </inertial>
  </link>

  <!-- Shoulder Joint (connects torso to upper arm) -->
  <joint name="shoulder_joint" type="revolute">
    <parent link="torso"/>
    <child link="upper_arm"/>
    <origin xyz="0.15 0 0.2" rpy="0 0 0"/>
    <axis xyz="1 0 0"/>
    <limit lower="-1.57" upper="1.57" effort="50" velocity="2.0"/>
  </joint>

  <!-- Forearm -->
  <link name="forearm">
    <visual>
      <geometry>
        <cylinder radius="0.03" length="0.25"/>
      </geometry>
      <origin xyz="0 0 -0.125" rpy="0 0 0"/>
      <material name="skin">
        <color rgba="0.9 0.8 0.7 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <cylinder radius="0.03" length="0.25"/>
      </geometry>
      <origin xyz="0 0 -0.125" rpy="0 0 0"/>
    </collision>
    <inertial>
      <mass value="0.5"/>
      <origin xyz="0 0 -0.125" rpy="0 0 0"/>
      <inertia ixx="0.005" ixy="0.0" ixz="0.0"
               iyy="0.005" iyz="0.0" izz="0.0005"/>
    </inertial>
  </link>

  <!-- Elbow Joint (connects upper arm to forearm) -->
  <joint name="elbow_joint" type="revolute">
    <parent link="upper_arm"/>
    <child link="forearm"/>
    <origin xyz="0 0 -0.3" rpy="0 0 0"/>
    <axis xyz="1 0 0"/>
    <limit lower="0" upper="2.35" effort="30" velocity="2.0"/>
  </joint>

  <!-- Hand -->
  <link name="hand">
    <visual>
      <geometry>
        <box size="0.08 0.04 0.12"/>
      </geometry>
      <origin xyz="0 0 -0.06" rpy="0 0 0"/>
      <material name="skin">
        <color rgba="0.9 0.8 0.7 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <box size="0.08 0.04 0.12"/>
      </geometry>
      <origin xyz="0 0 -0.06" rpy="0 0 0"/>
    </collision>
    <inertial>
      <mass value="0.2"/>
      <origin xyz="0 0 -0.06" rpy="0 0 0"/>
      <inertia ixx="0.001" ixy="0.0" ixz="0.0"
               iyy="0.001" iyz="0.0" izz="0.0005"/>
    </inertial>
  </link>

  <!-- Wrist Joint (connects forearm to hand) -->
  <joint name="wrist_joint" type="revolute">
    <parent link="forearm"/>
    <child link="hand"/>
    <origin xyz="0 0 -0.25" rpy="0 0 0"/>
    <axis xyz="0 1 0"/>
    <limit lower="-0.785" upper="0.785" effort="10" velocity="2.0"/>
  </joint>

</robot>
```

### Key Features:

*   **Inertial Properties**: Required for physics simulation (Gazebo, Isaac Sim)
*   **Collision Geometry**: Can differ from visual geometry for performance
*   **Joint Limits**: Realistic limits based on human anatomy
*   **Origin Offsets**: Position child links correctly relative to parents

## Visualizing URDF in RViz2

RViz2 is the standard visualization tool for ROS 2. To visualize your URDF:

```bash
# Install joint_state_publisher and robot_state_publisher
sudo apt install ros-humble-joint-state-publisher-gui ros-humble-robot-state-publisher

# Launch visualization
ros2 launch urdf_tutorial display.launch.py model:=path/to/your/humanoid.urdf
```

The `joint_state_publisher_gui` allows you to interactively move joints and see the robot's configuration in RViz2.

## XACRO: Simplifying Complex URDFs

For complex humanoid models, raw URDF files can become unwieldy. **Xacro** (XML Macros) extends URDF with:

*   **Variables**: Define reusable constants
*   **Macros**: Create parameterized templates (e.g., for limbs)
*   **Math Expressions**: Compute values dynamically

Example Xacro snippet:

```xml title="humanoid.urdf.xacro"
<?xml version="1.0"?>
<robot name="humanoid" xmlns:xacro="http://www.ros.org/wiki/xacro">

  <!-- Properties -->
  <xacro:property name="arm_length" value="0.3"/>
  <xacro:property name="arm_radius" value="0.04"/>
  <xacro:property name="arm_mass" value="1.0"/>

  <!-- Macro for an arm link -->
  <xacro:macro name="arm_link" params="name length radius mass">
    <link name="${name}">
      <visual>
        <geometry>
          <cylinder radius="${radius}" length="${length}"/>
        </geometry>
        <origin xyz="0 0 ${-length/2}" rpy="0 0 0"/>
        <material name="skin">
          <color rgba="0.9 0.8 0.7 1"/>
        </material>
      </visual>
      <inertial>
        <mass value="${mass}"/>
        <origin xyz="0 0 ${-length/2}" rpy="0 0 0"/>
        <inertia ixx="${mass*length*length/12}" ixy="0.0" ixz="0.0"
                 iyy="${mass*length*length/12}" iyz="0.0"
                 izz="${mass*radius*radius/2}"/>
      </inertial>
    </link>
  </xacro:macro>

  <!-- Use the macro -->
  <xacro:arm_link name="left_upper_arm" length="${arm_length}"
                  radius="${arm_radius}" mass="${arm_mass}"/>
  <xacro:arm_link name="right_upper_arm" length="${arm_length}"
                  radius="${arm_radius}" mass="${arm_mass}"/>

</robot>
```

Convert Xacro to URDF:

```bash
xacro humanoid.urdf.xacro > humanoid.urdf
```

## Adding Sensors to Humanoid Models

Humanoid robots often have cameras, IMUs, and force/torque sensors. Example camera:

```xml
<link name="camera_link">
  <visual>
    <geometry>
      <box size="0.02 0.05 0.02"/>
    </geometry>
    <material name="black">
      <color rgba="0 0 0 1"/>
    </material>
  </visual>
</link>

<joint name="camera_joint" type="fixed">
  <parent link="head"/>
  <child link="camera_link"/>
  <origin xyz="0.05 0 0" rpy="0 0 0"/>
</joint>

<!-- Gazebo plugin for camera -->
<gazebo reference="camera_link">
  <sensor type="camera" name="head_camera">
    <update_rate>30.0</update_rate>
    <camera name="head">
      <horizontal_fov>1.3962634</horizontal_fov>
      <image>
        <width>640</width>
        <height>480</height>
        <format>R8G8B8</format>
      </image>
      <clip>
        <near>0.02</near>
        <far>300</far>
      </clip>
    </camera>
    <plugin name="camera_controller" filename="libgazebo_ros_camera.so">
      <alwaysOn>true</alwaysOn>
      <updateRate>0.0</updateRate>
      <cameraName>robot/camera</cameraName>
      <imageTopicName>image_raw</imageTopicName>
      <cameraInfoTopicName>camera_info</cameraInfoTopicName>
      <frameName>camera_link</frameName>
    </plugin>
  </sensor>
</gazebo>
```

## Best Practices for Humanoid URDF Models

1. **Coordinate Frames**: Follow ROS conventions (x-forward, y-left, z-up)
2. **Joint Names**: Use clear, consistent naming (e.g., `left_shoulder_pitch`, `right_knee_roll`)
3. **Realistic Limits**: Base joint limits on human anatomy or hardware specs
4. **Center of Mass**: Ensure inertial origins are correct for stable simulation
5. **Collision vs. Visual**: Simplify collision geometry for performance
6. **Modular Design**: Use Xacro macros for symmetric limbs (left/right arms, legs)
7. **Testing**: Visualize in RViz2 and test in Gazebo before deploying to hardware
8. **Documentation**: Comment your URDF/Xacro files for maintainability

## Simulating Humanoids in Gazebo

Gazebo is the primary 3D robot simulator for ROS 2. To simulate your humanoid:

```bash
# Launch Gazebo with your robot
ros2 launch gazebo_ros gazebo.launch.py

# Spawn your robot
ros2 run gazebo_ros spawn_entity.py -entity humanoid -file path/to/humanoid.urdf
```

For full physics simulation, you'll need to add:

*   **Gazebo plugins**: For controllers, sensors, and actuators
*   **ros2_control**: For joint trajectory control
*   **Collision properties**: Friction, restitution coefficients

URDF is the foundation for modeling humanoid robots in ROS 2. By understanding links, joints, and how to structure complex kinematic chains, you can create accurate models for simulation, visualization, and control. Xacro further simplifies development for symmetric and modular humanoid designs.

Mastering URDF and Xacro is essential for anyone working with humanoid robotics, whether you're building custom platforms or working with existing research robots.
