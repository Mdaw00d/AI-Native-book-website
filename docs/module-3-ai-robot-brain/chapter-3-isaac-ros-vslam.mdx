## What is Isaac ROS?

**Isaac ROS** is NVIDIA's collection of **hardware-accelerated ROS 2 packages** that leverage GPU compute for robotics perception.

### Key Advantages

| Feature | CPU-Based (ORB-SLAM3, RTABMap) | Isaac ROS (GPU-Accelerated) |
|---------|--------------------------------|----------------------------|
| **Stereo Depth** | 5-10 FPS @ 640×480 | **60 FPS** @ 640×480 |
| **Feature Extraction** | 20-30ms (ORB features) | **&lt;5ms** (GPU accelerated) |
| **VSLAM Throughput** | 10-15 FPS | **60 FPS** |
| **CPU Usage** | 80-100% (multiple cores) | **&lt;10%** (offloaded to GPU) |
| **Power Efficiency** | High CPU power draw | Optimized for Jetson edge devices |

<Callout type="success" title="10x Performance Gain">
Isaac ROS achieves **60 FPS VSLAM** on edge hardware (Jetson AGX Orin) - impossible with CPU-only approaches. This enables real-time navigation for fast-moving humanoid robots.
</Callout>

## Step 1: Install Isaac ROS

Two installation methods: **Docker (recommended)** or **Native installation**.

### Method 1: Docker Installation (Recommended)

**Pros**: Pre-configured environment, no dependency conflicts
**Cons**: Docker overhead (~5% performance loss)

<CodeBlock language="bash" title="Install Docker and NVIDIA Container Toolkit">
{`# Install Docker
sudo apt update
sudo apt install -y docker.io
sudo systemctl enable --now docker
sudo usermod -aG docker $USER  # Add user to docker group
newgrp docker  # Apply group change

# Install NVIDIA Container Toolkit
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
sudo apt update
sudo apt install -y nvidia-docker2
sudo systemctl restart docker

# Verify GPU access in Docker
docker run --rm --gpus all nvidia/cuda:11.8.0-base-ubuntu22.04 nvidia-smi
# Should show your GPU`}
</CodeBlock>

<CodeBlock language="bash" title="Pull Isaac ROS Docker Image">
{`# Pull official Isaac ROS image (includes cuVSLAM, stereo depth)
docker pull nvcr.io/nvidia/isaac-ros/isaac-ros-dev-x86_64:release-2.0

# Create workspace directory
mkdir -p ~/isaac_ros_ws/src
cd ~/isaac_ros_ws

# Launch Isaac ROS development container
docker run --privileged --network host --gpus all \\
  -v ~/isaac_ros_ws:/workspaces/isaac_ros_ws \\
  -v /tmp/.X11-unix:/tmp/.X11-unix \\
  -e DISPLAY=$DISPLAY \\
  --name isaac-ros-dev \\
  -it nvcr.io/nvidia/isaac-ros/isaac-ros-dev-x86_64:release-2.0

# Inside container:
# /workspaces/isaac_ros_ws$ source /opt/ros/humble/setup.bash`}
</CodeBlock>

## Step 3: Deploy cuVSLAM

**cuVSLAM** is NVIDIA's GPU-accelerated Visual SLAM with loop closure detection.

### cuVSLAM vs ORB-SLAM3

| Feature | ORB-SLAM3 (CPU) | cuVSLAM (GPU) |
|---------|----------------|---------------|
| **Frame Rate** | 10-15 FPS | **60 FPS** |
| **Loop Closure** | 5-10 seconds | **&lt;3 seconds** |
| **Relocalization** | Slow | Fast (GPU feature matching) |
| **Map Optimization** | CPU-bound | GPU bundle adjustment |
| **Hardware** | Any CPU | Requires NVIDIA GPU |

<Callout type="info" title="When to Use Each">
- **cuVSLAM**: Real-time applications, edge deployment (Jetson), large-scale maps
- **ORB-SLAM3**: Research, CPU-only systems, visual-inertial fusion (cuVSLAM doesn't support IMU yet)
</Callout>

## Step 4: Visual Odometry Fallback

When VSLAM fails (texture-less walls, rapid motion), **visual odometry (VO)** provides fallback tracking.

### Configure VO Parameters

<CodeBlock language="yaml" title="vslam_params.yaml">
{`/**:
  ros__parameters:
    # Enable visual odometry fallback
    enable_observations_view: true
    enable_slam_visualization: true
    enable_debug_mode: false

    # VO fallback triggers
    min_num_features: 50  # Switch to VO if features &lt;50
    max_vo_drift_percentage: 2.0  # Max 2% drift over 100m

    # Feature detection
    num_cameras: 2  # Stereo pair
    min_quality_features: 200
    max_features_per_frame: 5000

    # Loop closure
    enable_loop_closure: true
    loop_closure_frequency: 1.0  # Check every 1 second
    loop_closure_min_score: 0.05`}
</CodeBlock>

<CodeBlock language="bash" title="Launch with custom parameters">
{`ros2 launch isaac_ros_visual_slam isaac_ros_visual_slam.launch.py \\
  params_file:=vslam_params.yaml`}
</CodeBlock>

## Step 6: Accuracy Validation

### 100m Trajectory Test

<CodeBlock language="python" title="validate_accuracy.py">
{`#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from nav_msgs.msg import Odometry
import numpy as np

class AccuracyValidator(Node):
    def __init__(self):
        super().__init__('accuracy_validator')
        self.subscription = self.create_subscription(
            Odometry,
            '/visual_slam/tracking/odometry',
            self.odom_callback,
            10
        )
        self.positions = []

    def odom_callback(self, msg):
        pos = msg.pose.pose.position
        self.positions.append([pos.x, pos.y, pos.z])

    def calculate_drift(self):
        if len(self.positions) < 2:
            return 0.0

        positions = np.array(self.positions)
        trajectory_length = np.sum(np.linalg.norm(np.diff(positions, axis=0), axis=1))

        # Assuming closed loop (return to start)
        drift = np.linalg.norm(positions[-1] - positions[0])
        drift_percentage = (drift / trajectory_length) * 100

        self.get_logger().info(f"Trajectory length: {trajectory_length:.2f}m")
        self.get_logger().info(f"Final drift: {drift*100:.2f}cm ({drift_percentage:.2f}%)")

        return drift

def main():
    rclpy.init()
    validator = Node('accuracy_validator')
    # Run for 100m trajectory...
    # Then call calculate_drift()
    rclpy.shutdown()

if __name__ == '__main__':
    main()`}
</CodeBlock>

<Callout type="success" title="Target Accuracy">
**Acceptable performance** (100m indoor trajectory):
- Position error: &lt;5cm
- Orientation error: &lt;2°
- Drift rate: &lt;0.05% of distance traveled
</Callout>

## Troubleshooting

### Issue 1: Low FPS (&lt;30 FPS)

**Symptoms**: VSLAM running slowly despite powerful GPU

**Fixes**:
<CodeBlock language="bash">
{`# 1. Check GPU isn't throttling
nvidia-smi
# Temp should be &lt;85°C, Perf should be "P0" (max performance)

# 2. Reduce image resolution (if using >640×480)
# Edit camera topics to publish 640×480 instead of 1080p

# 3. Disable visualization during benchmarking
ros2 param set /visual_slam enable_slam_visualization false`}
</CodeBlock>

### Issue 2: Tracking Lost Frequently

**Symptoms**: \`tracking_state: LOST\` in diagnostics

**Fixes**:
- **Increase lighting**: Ensure scenes have adequate light (>500 lux)
- **Add visual features**: Place textured objects in featureless areas
- **Slow camera motion**: Reduce robot velocity during fast turns
- **Enable VO fallback**: Ensure \`enable_observations_view: true\`

### Issue 3: High Drift (>10cm over 100m)

**Symptoms**: Robot returns to start but position estimate is far off

**Fixes**:
<CodeBlock language="yaml">
{`# Increase loop closure frequency
loop_closure_frequency: 2.0  # Check every 0.5 seconds

# Lower loop closure threshold (more aggressive matching)
loop_closure_min_score: 0.03  # Default 0.05

# Enable pose graph optimization
enable_pose_graph_optimization: true`}
</CodeBlock>
