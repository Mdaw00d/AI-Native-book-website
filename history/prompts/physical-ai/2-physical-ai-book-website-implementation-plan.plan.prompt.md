## Prompt

Planning:
Phase 1 — Set up a premium Docusaurus project with MDX v3, dark/light mode, minimal theme customizations, clean typography, and autogenerated sidebars for a book-style layout.
Phase 2 — Build global layout: persistent left sidebar with all modules/chapters, responsive mobile drawer, TOC, breadcrumbs, reading container, and a global progress bar for docs.
Phase 3 — Home page: Hero explaining Physical AI + its purpose, module overview, chapter previews, and CTA to start Module 1; smooth transitions, mobile-first, dark-mode optimized.
Phase 4 — Module 1 reading system: MDX-based chapters rendered with Docusaurus docs engine, TOC, reading progress, keyboard navigation, scroll restoration, copy buttons, code highlighting, and localStorage-based reading position.
Phase 5 — About + Contact pages: About page explaining capstone vision + teaching approach; Contact page with localStorage-driven waitlist form and success screen (no backend).
Phase 6 — Content: Write all four Module 1 MDX chapters—ROS 2 Middleware, Nodes/Topics/Services, Python-to-ROS bridging (rclpy), and URDF for humanoids—with consistent formatting and premium readability.
Phase 7 — Polish + deploy: Improve spacing, typography, dark mode, sidebar stability, mobile drawer performance, and navigation flow; test thoroughly and deploy via Vercel or GitHub Pages.

Important Decisions:
MDX over plain React (content-first, easy updates);
Docusaurus UI over custom Next.js (built-in docs engine + versioning);
localStorage over backend (MVP simplicity);
Multi-module-ready structure (scales as Physical AI grows);
Minimal theme overrides for best performance + clean design.
